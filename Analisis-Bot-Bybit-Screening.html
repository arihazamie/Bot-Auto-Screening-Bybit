Decorator _with_retry() menangani RateLimitExceeded, NetworkError, dan RequestTimeout secara otomatis dengan delay 1s → 2s → 4s. Pemisahan antara "retryable errors" dan "fatal errors" (BadSymbol, AuthError) sudah tepat.

OHLCV Cache + Leverage Cache dengan TTL

JSON File Storage Tidak Scalable & Rawan Race Condition

Seluruh storage menggunakan file JSON (signals.json, active_trades.json, dll). Satu global lock _lock dipakai untuk semua operasi. Saat thread count tinggi, ini menjadi bottleneck. Seiring waktu, file bisa tumbuh tak terbatas (tidak ada cleanup). Tidak ada query/index — setiap lookup harus baca seluruh file.

# ✅ Setelah fix — database.py memakai SQLite WAL mode:
import sqlite3
conn = sqlite3.connect('data/bot.db', check_same_thread=False)
# WAL: concurrent reads, serialised writes, crash-safe, ada index per kolom
conn.execute("PRAGMA journal_mode=WAL")
FIXED ✓

technicals.py Memanggil df.dropna(inplace=True) Secara Langsung

Fungsi get_technicals() memanggil df.dropna(inplace=True) yang memodifikasi DataFrame secara in-place dan mengembalikan nilai. Ini mengubah panjang data secara tak terduga, berpotensi menyebabkan index out-of-bounds di modul lain yang mengasumsikan panjang DataFrame konstan. Dengan pandas baru, inplace juga menghasilkan SettingWithCopyWarning.

# ✅ Setelah fix — DataFrame baru, index di-reset:

4. Saran Strategi Trading
Peningkatan kualitas sinyal dan akurasi entry

🎯 Entry Point

  • Gunakan current bid/ask sebagai entry utama, bukan kalkulasi fib yang sering meleset
  • Limit order zone: Set pending order di range 0.1–0.3% dari harga saat ini, bukan 3% drift
  • Tambahkan konfirmasi candle: Tunggu satu candle close setelah pattern terbentuk sebelum entry

⚠️ Stop Loss

  • ATR × 1.5 sudah tepat, tapi tambahkan floor minimum: SL minimum 0.5% dari entry
  • Aktifkan require_sl_beyond_swing: Di config default ini false, seharusnya true
  • Hindari round numbers: Tambahkan buffer 0.1 ATR di luar swing untuk menghindari stop hunt

📉 Divergence Detection

  • StochRSI divergence saat ini hanya pakai indeks pivot terakhir — tambahkan look-forward confirmation
  • MACD histogram divergence sudah dihitung tapi tidak dipakai dalam scoring — integrasikan
  • RSI divergence (14) sebaiknya ditambahkan sebagai secondary confirmation

🏦 SMC Enhancement

  • Premium/Discount zones: Cek apakah entry berada di discount (Long) atau premium (Short) menggunakan range 50% level
  • Inducement detection: Identifikasi liquidity inducement sebelum OB untuk entry yang lebih presisi
  • Multi-timeframe OB: Cek OB pada 4H dan 1D juga untuk confluence yang lebih kuat

🔢 Scoring System

  • Bobot scoring tidak merata: SMC bisa max 10+, quant hanya 2-4. Normalisasi bobot agar lebih seimbang
  • Total score threshold: Tambahkan filter minimum total score sebelum signal dikirim
  • Score historis: Track akurasi setiap pattern/score combination untuk auto-tuning threshold

📊 Market Filter Tambahan

  • Fear & Greed Index: Hindari Long saat Extreme Fear, hindari Short saat Extreme Greed
  • Open Interest change: Rising OI + Rising Price = lebih kuat untuk Long
  • Session filter: Saat ini hanya UTC hour window — tambahkan volatility filter berdasarkan ATR percentile

Position Sizing Berbasis Volatilitas (ATR-Based)

Saat ini risk_percent flat 1% untuk semua coin. Coin dengan volatilitas tinggi (ATR besar) seharusnya mendapat alokasi lebih kecil. Implementasikan: adjusted_risk = risk_percent × (target_atr_pct / actual_atr_pct) dengan batas min 0.5% dan max 2%.
